2.  โปรแกรม 1 — en-th-word-swap
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  จุดประสงค์:
    แก้ปัญหาพิมพ์ผิดภาษาโดยไม่ตั้งใจ สองทิศทาง:
    - Direction A: พิมพ์ไทยทั้งที่ตั้งใจพิมพ์อังกฤษ → ดนสกำพ → suggest "folder"
    - Direction B: พิมพ์อังกฤษทั้งที่ตั้งใจพิมพ์ไทย → df → suggest "กด"

  CLI: ENTHswap
    ENTHswap             — รัน daemon
    ENTHswap --stop      — หยุด daemon
    ENTHswap --status    — ดูสถานะ
    ENTHswap --setting   — ตั้งค่า
    ENTHswap --install   — ติดตั้ง LaunchAgent
    ENTHswap --uninstall — ถอนติดตั้ง
    ENTHswap --update    — อัปเดต corpus

  ไฟล์และหน้าที่:
    main.py        CGEventTap + state machine + logic หลัก
    keyboard_map.py  Kedmanee EN↔TH mapping table + word validation
    overlay.py     NSPanel floating popup (SuggestionOverlay)
    settings.py    ค่าตั้งต่าง ๆ → cache/settings.json
    updater.py     อัปเดต corpus รายวัน + UserTracker (personal exclusion)
    ENTHswap       CLI entry point
    install.sh     ติดตั้ง LaunchAgent + ถาม auto-start
    uninstall.sh   ถอนติดตั้ง

  การทำงาน (flow):
    1. CGEventTap intercept ทุก kCGEventKeyDown ก่อนแอปอื่น
    2. สะสม char ใน _buf ตรวจ mode (thai / en)
    3. กด Space → ตรวจ _get_suggestion()
       - Direction A: thai_to_english() → ตรวจ /usr/share/dict/words
       - Direction B: english_to_thai() → ตรวจ PyThaiNLP corpus
    4. ถ้ามี suggestion → แสดง SuggestionOverlay → รอ confirm
    5. Accept (Space/Enter) → ลบ char เดิม → inject ข้อความใหม่
    6. Dismiss (Esc/BS) → บันทึกใน UserTracker

  Settings ที่ปรับได้:
    accept_key  : space | enter  (ปุ่มยืนยัน)
    dismiss_bs  : false | true   (Backspace เป็น dismiss หรือไม่)
    auto_start  : true | false   (เริ่มอัตโนมัติหลัง login)

  Tests: tests/ (69 tests)
    test_keyboard_map.py  — mapping table, Direction A/B, Thai validation
    test_settings.py      — persistence, defaults, merge logic
    test_suggestion.py    — suggestion logic
    test_updater.py       — UserTracker, JSON helpers, cache loading

  Dependencies:
    pyobjc, pythainlp

  Cache files (gitignored):
    cache/settings.json     — user settings
    cache/user_learned.json — personal word exclusion list
    cache/en_internet.txt   — Wiktionary internet slang cache
    cache/meta.json         — update timestamps
    cache/daemon.pid        — PID file

3.  โปรแกรม 2 — num-to-text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  จุดประสงค์:
    พิมพ์ตัวเลข กด Enter → popup แสดงตัวเลขเป็นคำอ่านทุกภาษาในระบบ
    รองรับ normal mode และ financial (currency) mode

  CLI: ntt
    ntt              — รัน daemon
    ntt --stop       — หยุด daemon
    ntt --status     — ดูสถานะ
    ntt --setting    — ตั้งค่า auto-start
    ntt --install    — ติดตั้ง LaunchAgent
    ntt --uninstall  — ถอนติดตั้ง
    ntt --update     — อัปเดต packages

  ไฟล์และหน้าที่:
    main.py      CGEventTap + state machine + popup control
    converter.py  ตัวเลข → คำ (Thai/Japanese/Chinese custom, อื่น ๆ ใช้ num2words)
    languages.py  อ่าน AppleLanguages → map locale → num2words code
    overlay.py   NSPanel floating popup พร้อม clickable rows (NumberOverlay)
    settings.py  auto_start setting → cache/settings.json
    ntt          CLI entry point
    install.sh   ติดตั้ง LaunchAgent + ถาม auto-start
    uninstall.sh ถอนติดตั้ง

  การทำงาน (flow):
    1. CGEventTap intercept ทุก kCGEventKeyDown
    2. สะสมตัวเลข/จุดทศนิยม/ลบ ใน _buf
    3. กด Enter → ตรวจ regex r"^-?\d+(\.\d+)?$"
    4. _build() → แปลงทุก language → (normal_rows, financial_rows)
    5. แสดง NumberOverlay พร้อมภาษาจาก AppleLanguages
    6. ผู้ใช้เลือก: 1-5 เลือกแถว, Enter=แถวแรก, Tab=toggle financial, Esc/BS=ปิด
    7. _select_row() → ลบตัวเลขเดิม → inject ข้อความ

  Controls ใน popup:
    1–5        เลือกภาษา → copy + พิมพ์ลงแอป
    Enter      เลือกภาษาแรก
    Tab        toggle normal ↔ financial mode
    Esc/BS     dismiss
    คลิก       copy ไป clipboard (แต่ไม่พิมพ์ลงแอป ← ดู bugs)
    Ctrl+Opt+F global toggle financial mode default

  Languages รองรับ:
    th (ภาษาไทย), en (English), de (Deutsch), ja (日本語),
    zh (中文), fr (Français), es (Español), ko (한국어), ru (Русский)

    Chinese  — รองรับ 万/亿, 十X shorthand, 零 padding

4.  สถาปัตยกรรมร่วมของโปรแกรม
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Pattern ที่ใช้เหมือนกัน:
    - CGEventTap: kCGSessionEventTap, kCGHeadInsertEventTap
    - _SYNTH_MARK tag (kCGEventSourceUserData): ระบุ synthetic event แทนการนับ
      counter (en-th, num — เปลี่ยนจาก _synth_pending ตอนแก้ BUG-9)
    - _claim_singleton(): kill instance เก่า + เขียน PID ภายใต้ fcntl.flock
      (กัน race ตอนแก้ BUG-10; เดิมเป็น _kill_existing ไม่มี lock)
    - _in_password_field() + _in_terminal_app(): กัน inject ลง password/terminal
      (en-th, num — เพิ่ม terminal guard ตอนแก้ BUG-1)
    - atexit PID cleanup
    - NSPanel: NSNonactivatingPanelMask + NSFloatingWindowLevel
    - NSWindowCollectionBehaviorCanJoinAllSpaces: ขึ้นทุก Desktop
    - Retry loop 30s เมื่อไม่มี Accessibility permission
    - daemon=True threads ทั้งหมด

  shared concepts แต่ implementation ต่างกัน:
    overlay.py    — ทั้งสามมี; SuggestionOverlay/NumberOverlay สร้างครั้งเดียว
                    แล้ว reuse (NumberOverlay แก้ให้ reuse ตอน BUG-6)
    settings.py   — en-th หลาย field, num มีแค่ auto_start, morse ไม่มี settings.py
    การ inject    — en-th/num inject ข้อความลงแอปอื่น; morse ไม่ inject
                    (ใช้ CGEventTap แค่ hotkey Option+M)

  [BUG-9] _synth_pending race condition (async CGEventPost)
    ไฟล์: main.py ทั้งสองโปรแกรม
    CGEventPost async — synthetic events อาจถึง callback หลัง
    real keypress อื่น ๆ ทำให้ counter อาจ mismatch
    ผลกระทบ: rare แต่อาจทำให้ real keystroke ถูก suppress ผิด
    Comment ใน code ยืนยัน known issue แต่ยังไม่แก้

  [BUG-10] PID file ไม่ lock (race condition)
    ไฟล์: main.py _kill_existing() ทั้งสองโปรแกรม
    อ่าน/เขียน PID file ไม่ได้ใช้ file lock
    ถ้า start 2 instances พร้อมกัน อาจเกิด race condition
    ในทางปฏิบัติ LaunchAgent ไม่ start ซ้ำ แต่อาจเกิดจาก manual run
